# DevOps, CI/CD, Docker та Розгортання

## 1. Контейнеризація (Docker)

Проєкт використовує Docker для ізоляції кожного мікросервісу та бази даних. 
- Кожен сервіс має власний `Dockerfile`.
- Кореневий `docker-compose.yml` об'єднує всі сервіси (auth, order, payment, product, rating, gateway) та інфраструктурні компоненти (PostgreSQL, Redis, RabbitMQ).

**Найкращі практики мультиконтейнерних систем, впроваджені в проєкті:**
- **Єдина база даних з ізольованими схемами/базами:** Використовується єдиний контейнер PostgreSQL з різними базами (`auth_db`, `product_db`, тощо) через скрипт `init-db.sh`. Це економить ресурси на локальному середовищі, зберігаючи логічну ізоляцію.
- **Управління змінними середовища:** Використовуються `.env` файли.
- **Healthchecks & Depends_on:** У `docker-compose.yml` налаштовано healthchecks для бази даних. Сервіси чекають повної готовності БД завдяки `depends_on: db: condition: service_healthy`. Це запобігає падінню сервісів під час старту.
- **Внутрішня мережа Docker:** Сервіси взаємодіють всередині `ecommerce_network` за внутрішніми DNS-іменами (`productservice:8003`), що гарантує безпеку (не всі порти відкриті назовні).
- **API Gateway:** Nginx виступає точкою входу, проксіюючи запити до відповідних сервісів, приховуючи їх внутрішні порти.

## 2. Інструкція з розгортання

### Локальне розгортання:

1. Клонувати репозиторій.
2. Скопіювати файл середовища: `cp .env.example .env` та налаштувати змінні (JWT_SECRET, DATABASE_URL тощо).
3. Запустити всі контейнери у фоновому режимі:
   ```bash
   docker-compose up -d --build
   ```
4. Frontend доступний за адресою `http://localhost:3000` (або порт з `vite.config`).
5. Backend API Gateway доступний за адресою `http://localhost:8000`.

### Міграції баз даних:
SQLAlchemy автоматично створює таблиці при старті сервісів (`Base.metadata.create_all(bind=engine)`). Для production рекомендується використання Alembic для версіонування міграцій.

## 3. Налаштування CI/CD

В проєкті налаштовано **GitHub Actions** (`.github/workflows/ci.yml`).

Конвеєр складається з наступних кроків:
1. **Checkout:** Завантаження коду.
2. **Setup Python / Node.js:** Встановлення середовища.
3. **Install Dependencies:** Встановлення залежностей через `pip` та `npm`.
4. **Linting & Formatting:** Перевірка якості коду (flake8, black, ESLint).
5. **Testing:** Запуск `pytest` для бекенду та юніт-тестів для фронтенду.
6. **Docker Build Check:** Перевірка, що всі Docker-образи успішно збираються.

## 4. Стратегія розгалуження (Branching Instructions)

Використовується спрощений GitFlow:
- `main` - продакшн гілка. Заборонено пушити напряму.
- `develop` - гілка для розробки та інтеграції фіч.
- `feature/*` - гілки для нових функцій (наприклад, `feature/add-repository-pattern`).
- `bugfix/*` - виправлення помилок.

**Процес:** 
Розробник створює `feature` гілку від `develop`. Після завершення розробки створюється Pull Request. CI-пайплайн автоматично проганяє тести. Після код-рев'ю гілка зливається в `develop`.
